二、常用 Markdown 语法

▶　标题

使用 # 表示。一级标题是一个 #，二级标题是两个 #，依此类推，最多到六级标题。

Markdown

复制

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

▶　文字样式

  • 斜体：使用一个星号 * 或下划线 _ 包裹文字，例如：斜体 或  斜体
  • 粗体：使用两个星号 ** 或下划线 __ 包裹文字，例如：粗体 或  粗体
  • 粗斜体：使用三个星号 *** 或下划线 ___ 包裹文字，例如：粗斜体 或  粗斜体
  • 删除线：使用两个波浪线 ~~ 包裹文字，例如：删除线
  • 代码：使用反引号 `` 包裹文字，例如： 代码

▶　列表

  • 无序列表使用星号 *、加号 + 或减号 - 作为列表标记： 

    * 项目1
* 项目2
* 项目3

有序列表使用数字和一个点 (.)：
1. 项目1 2. 项目2 3. 项目3

  • 列表嵌套：在一个列表项中开始另一个列表来创建嵌套的列表。

    1. 项目1
   - 子项目1
   - 子项目2
2. 项目2
   - 子项目1
   - 子项目2

引用可以被嵌套：

>

▶　代码和代码块

在 Markdown 中，你可以使用反引号 (` ) 来格式化代码。

这里反引号 (` ) 可以使用 1-3 个，用在不同场景：

1、单反引号
对于内联代码，你可以使用单个反引号。例如：

这是一个 `内联代码` 示例。

这是一个 内联代码 示例。

2、双反引号

如果你要表示为代码的单词或短语中包含一个或多个反引号，则可以通过将单词或短语包裹在双反引号 (` ` ) 中。例如：

 ``假如我有 包含`单引号` 的语句需要显示。``

假如我有 包含`单引号` 的语句需要显示。

3、三反应号
对于代码块，你可以使用三个反引号。也可以在开头的三个反引号后面添加语言标识符，以启用语法高亮。
例如：

```python
def hello_world():
    print("Hello, world!")
```

使用反斜杠 (\) 转义特殊字符：

▶　内联 HTML

你可以在 Markdown 文档中直接使用 HTML。

<div style="text-align:center">这是一段居中的文本。</div>

▶　锚点

可以创建链接到同一文档的其他部分的锚点。

[回到顶部](#标题)

▶　脚注

这是一段带有多个脚注的文本[^1][^2]。

[^1]: 这是第一个脚注的内容。

[^2]: 这是第二个脚注的内容。
    它实际上包含了两行。

在这个例子中，[^1] 是对脚注的引用，[^1]: 这是脚注的内容。 
标识符可以是数字或单词，但不能包含空格或制表符。标识符仅将脚注参考与脚注本身相关联-在输出中，脚注按顺序编号。
脚注代号可以随便命名，不过推荐使用数字序号。脚注代号+∶+空格＋脚注内容

这是一段带有多个脚注的文本[1]内容[2]。

▶　任务列表（Tasks）

可以通过 - [ ] 来创建一个未完成的任务，通过 - [x] 来创建一个已完成的任务。例如：

- [ ] 这是一个未完成的任务。
- [x] 这是一个已完成的任务。
  • 这是一个未完成的任务。
  • 这是一个已完成的任务。

三、Obsidian 特有语法

▶　特有链接 & 图片

和基本语法功能类似，只不过换了标志，对比： ▶ 链接 & 图片

Obsidian 通过使用双方括号 [[ ]] 实现的。

  • 链接：[[]]
  • 图片：`![[]]

链接：

查看我的笔记 [[My Note]]。

| 指定显示的文本
# 可以链接到标题输入
^ 链接文本块输入

当你点击这个链接时，Obsidian 将会打开 "工具使用" 这个笔记的“主题设置”这一章。
如下： 主题设置

文本块引用：

Obsidian 支持跨文件的块引用。你可以通过对特定文本块的引用，来在另一个文件中引用这个块。

首先，你需要在你想要引用的块后面添加一个 ^ 和一个标识符。例如： 

这是我想引用的文本块。^ab12cd

然后，在另一个文件中，你可以通过以下方式引用这个块：

这是我对 [[My Note#^ab12cd]] 的引用。

这将会创建一个到 "My Note" 笔记中 "block 1" 块的链接。

按照 ctrl 鼠标悬浮可以显示内容：测试引用块

嵌入笔记：

你可以在一个笔记中嵌入另一个笔记的内容。这是通过在链接前加上一个感叹号 ! 实现的。例如：

这是嵌入的笔记内容：![[My Note]]

这将会在当前笔记中显示 "My Note" 这个笔记的所有内容。

如果文件不存在会提示：
"MyNote.md" 未创建，点击以创建。

四、标准规范

一）数字序号用类别

1、数字序号的级别顺序为：

层级 表示方法 其他方法
第一层 一、二、三、
第二层 （一）（二）（三）
第三层 1.2.3.
第四层 （1）（2）（3）
第五层 ① ② ③ 1）2）3）
第六层 A. B. C. (A) (B) (C)
第七层 a. b. c. (a) (b)（c)

2、文稿的层次序号：

文稿的层次序号分理科与人文两种。在实际应用的过程中，一般都不超过四级。
理科类一律用阿拉伯数字连续编号，人文类一般沿用公文层次序号。

层次 人文 理科
第一层 一、二、三、 1.
第二层 （一）（二）（三） 1.1
第三层 1.2.3. 1.1.1
第四层 （1）（2）（3） 1.1.1.1
第五层 ① ② ③
常见序号的错误用法
1、 A、 （一）、 (1)、 ①、 第一： 第一、
2、 B、 （二）、 （2）、 ②、 第二： 第二、
3、 C、 （三）、 （3）、 ③、 第三： 第三、
常见序号的正确用法
1. A. （一） (1) ① 第一， 首先，
2. B. （二） （2） ② 第二， 其次，
3. C. （三） （3） ③ 第三， 再次，

五、适用于自己的文档规范

一）序号规范

每次在写 md 文档时，总是会花大量时间去考虑笔记的排版问题。
其实更应该将经历注重在学习和记录总结知识方面。因此特定此文档为今后的文档做出规范。加新文档时可直接复制。

Markdown 中因为 1. 就是一个序号，所以这边使用 1、来代替。
并且由于笔记原因，层次结构复杂：
以及标题默认为文件名称
所以这里层级可以延伸至

即：

一级标题：最好和文档名称文字保持一致，如果有多个一级 ★ 一级标题 ★
二级标题：无序的使用符号：▷　，如果需要数字标题使用 一、二、三、四、
三级标题：无序的使用符号：▶　，如果需要数字标题使用 一）二） 三）四）
四级标题：无序的使用符号：❈　，如果需要数字标题使用 1、 2、3、4、
五级标题：无序的使用符号：○　，如果需要数字标题使用 1）2）3）4） 或 ① ② ③ ④ ⑤ ⑥ ⑦ ⑧ ⑨
六级标题：无序的使用符号：●　，如果需要数字标题使用 ❶ ❷ ❸ ❹ ❺ ❻ ❼ ❽ ❾ ❿, 其中需要加空格

如果不是大类，三级标题转四，也不算违规，一） 变成 1、，更便于阅读，但目前而言，这种情况是极少数的。
因为 markdown 只支持到 6 级标题。

6 级以下：
七级标题： A. B. C. 或 (A) (B) (C)
七级标题： a. b. c. 或 (a) (b) (c)

如果标题较少，并且无序，不管几级标题也可忽略符号，方便查阅，避免目录全是符号。

同时尽量少用标题，虽然可以生成目录头，但是有些没有明显意义的不需要标题，但是加上符号阅读时方便即可。

以下是样例代码：

#### 1、测试四级标题
####  ❈　测试四级标题

##### ① 测试五级标题

##### ○　测试五级标题

###### ❶ 测试六级标题
###### ●　测试六级标题

二）附件存放规范

附件统一存放位置：./assets/${filename}
图片命名规范: Img-YYYYMMDD_HHmmss
图片格式最好为：. PNG 这种支持高清方式

文章内所有链接统一使用标准 markdown 格式，防止使用 typora 使连接失效。
如果已经有 obisidian 的链接了，单文件可以直接选中已有文件，只要设置对了会自动变成 markdown 的原本格式。
